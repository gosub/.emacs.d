* Overview

This guide covers two areas:

1. Experimental music recipes using CDP8 generators and effects chained
   as nodes in ~cdp8-mode~.
2. Standard DAW workflows (mixing, envelopes, arranging, drum programming,
   mastering) implemented via CDP8 and SoX together.

The mode represents processing chains as typed cables:
- =a1.wav= =a2.wav= … audio
- =s1.ana= =s2.ana= … spectral analysis (phase vocoder)
- =p1.frq= =p2.frq= … pitch data
- =b1.brk= =b2.brk= … breakpoint (time/value text files)
- =e1.env= =e2.env= … binary envelope data

Nodes are created with =n=, run with =r= or =R=, and inspected with =RET=.
Edit a node's parameters with =e=, its raw command string with =E=.

** SoX integration

Load the optional SoX module and call =cdp8-sox-setup= once after the session is open:

#+begin_src emacs-lisp
(require 'cdp8-sox)          ; add to your init or eval in *scratch*
(cdp8-sox-setup)             ; merges 21 SoX effects into the running session
#+end_src

After setup, SoX effects (=sox reverb=, =sox norm=, =sox compand=, …) appear in the
=n= command list and in the =RET= effect menu exactly like CDP8 commands.
The schema system handles the inverted SoX argument order (=sox infile outfile effect params=)
automatically — parameters are prompted field-by-field just like CDP8 effects.

** Text data files (breakpoint / sequence files)

Many CDP8 parameters are text files: breakpoint envelopes, frequency tables,
grain timing lists, etc.  When prompted for such a parameter the mode offers two choices:

- =y= — pick an existing file with =read-file-name=
- =n= — open a compose window pre-populated with a format comment; write the data,
  then =C-c C-c= to confirm or =C-c C-q= to cancel

The composed file is saved to the session directory so it persists with the project.

* Experimental Music: 20 Composition Recipes

Each recipe is a node chain. Indentation shows data flow. Parameters
shown are starting points — explore the ranges.

** 1. Spectral Freeze Drone

Freeze a single analysis window and stretch it to a texture.

#+begin_example
  [generator]  synth wave   → a1.wav
               (sine or complex source, a few seconds)
  [effect]     pvoc anal    → s1.ana
  [effect]     spec grab    → s2.ana    (time=0.5)
  [effect]     spec magnify → s3.ana   (dur=30.0)
  [effect]     pvoc synth   → a2.wav
#+end_example

Result: an infinite-seeming spectral drone from a snapshot of the source.
Vary =time= in =spec grab= to select different spectral moments.

** 2. Granular Cloud Diffusion

Scatter grains in time, then blur the spectral result.

#+begin_example
  [generator]  synth wave           → a1.wav
  [effect]     grain r_extend       → a2.wav   (time=10.0)
  [effect]     pvoc anal            → s1.ana
  [effect]     blur scatter         → s2.ana   (keep=2)
  [effect]     pvoc synth           → a3.wav
#+end_example

Result: granular diffusion converted into a smeared, hazy spectral texture.

** 3. Stochastic Spectral Walk

Use blur drunk (random walk through time windows) to create
unpredictable morphing textures.

#+begin_example
  [source]     any recorded sound   → a1.wav
  [effect]     pvoc anal            → s1.ana
  [effect]     blur drunk           → s2.ana   (range=8, duration=20.0)
  [effect]     pvoc synth           → a2.wav
#+end_example

The spectral content from different moments of the source is combined
unpredictably. Each run produces a different result.

** 4. Cross-Synthesis (Spectral Hybridisation)

Impose the spectral envelope of one sound onto the partials of another.

#+begin_example
  [source A]   any sound A          → a1.wav
  [source B]   any sound B          → a2.wav
  [effect]     pvoc anal (a1)       → s1.ana
  [effect]     pvoc anal (a2)       → s2.ana
  [effect]     specross (s1, s2)    → s3.ana
               (partials=8, tuning=440)
  [effect]     pvoc synth           → a3.wav
#+end_example

Result: the rhythm/attack of A with the spectral colour of B (or a blend).

** 5. Harmonic Distortion Chain

Stack harmonic distortion and filter it for controlled timbral enrichment.

#+begin_example
  [generator]  synth wave           → a1.wav   (simple sine)
  [effect]     distort harmonic     → a2.wav   (harmonics-file with partials 1-5)
  [effect]     filter lohi          → a3.wav   (lofrq=200, hifrq=4000)
  [effect]     modify loudness      → a4.wav
#+end_example

Result: rich overtone structures from a clean source.

** 6. Spectral Gate and Freeze

Gate quiet partials, then freeze and stretch the survivors.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     pvoc anal            → s1.ana
  [effect]     spec gate            → s2.ana   (threshold=0.02)
  [effect]     hilite band          → s3.ana   (centrefrq=440, bandwidth=50)
  [effect]     spec magnify         → s4.ana   (dur=8.0)
  [effect]     pvoc synth           → a2.wav
#+end_example

The gate removes quiet partials; hilite band isolates a frequency region;
magnify extends the surviving spectrum into a sustained tone.

** 7. Granular Repitch Scatter

Use grain operations to scatter a sound across pitch space.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     grain repitch        → a2.wav   (transpmultfile with random ratios)
  [effect]     distort average      → a3.wav   (cyclecnt=4)
  [effect]     filter phasing       → a4.wav
#+end_example

Result: the source broken into randomly repitched grains, then averaged
and phase-modulated into a chorus-like texture.

** 8. Brownian Frequency Motion

Generate noise with Brownian frequency drift.

#+begin_example
  [generator]  brownian motion      → a1.wav
               (mode=1, plo=80, phi=1200, step=50, dur=20)
  [effect]     distort envel        → a2.wav
  [effect]     filter variable      → a3.wav
#+end_example

The Brownian motion generator produces a sinusoidal tone that drifts
randomly in frequency — an inherently organic, unpredictable oscillator.

** 9. Cantor Set Rhythmic Gating

Apply the fractal Cantor set to punch recursive silence holes into audio.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     cantor set           → a2.wav
               (mode=1, holesize=0.5, depth-trig=4)
  [effect]     pvoc anal            → s1.ana
  [effect]     blur chorus          → s2.ana
  [effect]     pvoc synth           → a3.wav
#+end_example

Nested silences at multiple time scales, then spectrally blurred.
Produces rhythmically structured, fragmented textures.

** 10. Spectral Sharpening via Focus Accumulation

Focus accumulation re-emphasises spectral peaks over time.

#+begin_example
  [source]     any tonal sound      → a1.wav
  [effect]     pvoc anal            → s1.ana
  [effect]     focus accu           → s2.ana   (acuity=6.0)
  [effect]     blur avrg            → s3.ana   (N=7)
  [effect]     pvoc synth           → a2.wav
#+end_example

=focus accu= sharpens spectral peaks; =blur avrg= then softens the result,
creating a resonant, metallic texture from any pitched source.

** 11. Iterative Distort Reform

Iterative reforming of waveform cycles for complex timbres.

#+begin_example
  [generator]  synth wave           → a1.wav   (complex waveform mix)
  [effect]     distort reform       → a2.wav   (mode=2)
  [effect]     distort reform       → a3.wav   (mode=3)
  [effect]     filter sweeping      → a4.wav
#+end_example

Chaining multiple =distort reform= calls builds up cycle-level
distortions that no single pass can produce.

** 12. Click Rhythm Machine

Use clicknew to generate a precise timed click sequence, then process it.

#+begin_example
  [generator]  clicknew             → a1.wav
               (clicks=8, clicktimes_datafile with beat times in seconds)
  [effect]     grain reverse        → a2.wav   (len=0.05)
  [effect]     modify revecho       → a3.wav
#+end_example

Clicknew places a click at each time listed in the data file; grain
reversal turns the clicks into small granular pops; revecho builds a
rhythmic wash. BPM 120, 4/4: times are 0.0, 0.5, 1.0, 1.5, …

** 13. Waveform Sculpting (Additive → Spectral)

Build a custom spectrum additively, then inject spectral noise.

#+begin_example
  [generator]  synth wave           → a1.wav
               (complex partials: mix modes 1-4)
  [effect]     pvoc anal            → s1.ana
  [effect]     blur noise           → s2.ana   (noise=0.3)
  [effect]     spec gain            → s3.ana   (gain=1.5)
  [effect]     pvoc synth           → a2.wav
#+end_example

Additive synthesis as a starting material for spectral noise injection.

** 14. Granular Time Reversal Cloud

Reverse grains at multiple scales for a backwards-texture effect.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     grain reverse        → a2.wav   (len=0.1)
  [effect]     grain reverse        → a3.wav   (len=0.3)
  [effect]     combine mean (a2, a3) → a4.wav
#+end_example

Two grain-reversed streams at different grain sizes averaged together.
Produces ambiguous temporal directionality.

** 15. Crystal Resonator

Crystal rotate for rotating spectral emphasis, then focus folding.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     crystal rotate       → a2.wav
  [effect]     pvoc anal            → s1.ana
  [effect]     focus fold           → s2.ana   (lofrq=100, hifrq=1000)
  [effect]     pvoc synth           → a3.wav
#+end_example

Crystal rotation shifts partials in phase; focus folding reflects high
partials back down, compressing the spectrum.

** 16. Spectral Blur Depth Map

Layer multiple blur amounts and subtract to isolate mid-timescale motion.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     pvoc anal            → s1.ana
  [effect]     blur blur (s1)       → s2.ana   (blurring=2)
  [effect]     blur blur (s1)       → s3.ana   (blurring=20)
  [effect]     specnu subtract (s2, s3) → s4.ana
  [effect]     pvoc synth           → a2.wav
#+end_example

Subtracting a heavily blurred version from a lightly blurred one isolates
mid-timescale spectral motion — a spectral "edge detector" in time.

** 17. Fractal Spectral Distortion

Use distort fractal for self-similar timbral complexity.

#+begin_example
  [source]     any sound            → a1.wav
  [effect]     distort fractal      → a2.wav   (scaling=4, loudness=1.0)
  [effect]     pvoc anal            → s1.ana
  [effect]     spec gate            → s2.ana   (threshold=0.01)
  [effect]     pvoc synth           → a3.wav
#+end_example

Fractal distortion adds self-similar copies of the waveform at different
scales. Spectral gating then removes the noise floor.

** 18. Ceracu Cycle Morphology

Ceracu repeats and stretches individual waveform cycles for organic timbres.

#+begin_example
  [source]     any short pitched sound → a1.wav
  [effect]     ceracu ceracu        → a2.wav
               (cyclcnts=10, mincycdur=0.1, outdur=20.0)
  [effect]     strans multi         → a3.wav   (speed=0.5)
  [effect]     modify space         → a4.wav
#+end_example

Ceracu analyses the source into waveform cycles and stretches each one
independently — a form of additive time-stretching. =strans multi= then
adds spectral transposition; =modify space= adds spatial movement.

** 19. Chirikov Map Modulator

Chirikov generates chaotic oscillations from a standard map.

#+begin_example
  [generator]  chirikov chirikov    → a1.wav
               (mode=1, frq=200, damping=0.5, dur=10.0)
  [effect]     distort multiply (a1, a1) → a2.wav   (self-ring-mod)
  [effect]     pvoc anal            → s1.ana
  [effect]     blur scatter         → s2.ana   (keep=2)
  [effect]     pvoc synth           → a3.wav
#+end_example

Chaotic frequency trajectories self-ring-modulated, then spectrally
scattered. Produces dense inharmonic textures.

** 20. Formant Singing Synthesis

Synthesise vowel formants and animate them with envelope data.

#+begin_example
  [generator]  synth spectra        → a1.wav
               (harmonic series partials, duration=4.0)
  [effect]     repitch vowels       → a2.wav
               (vowel-data = "a e i o u" sequence)
  [effect]     modify loudness      → a3.wav
  [effect]     modify revecho       → a4.wav
#+end_example

Harmonic source shaped by vowel formants over time, giving the impression
of a singing voice.

* Standard DAW Workflows

** Mixing

*** Basic Stereo Mix (CDP8)

CDP8's =submix= family handles basic DAW-style mixing.

#+begin_example
Create a mix file:
  submix atstep  (place two files at timed steps → outmixfile)

Add tracks to it:
  submix addtomix  (add more sound files to an existing mix)

Apply level faders:
  submix attenuate  (scale gain across mix entries)

Render to audio:
  submix getlevel  (preview peak levels)
  combine sum   or  combine mean   (render multi-track mix down)
#+end_example

*** Gain and Normalisation (SoX)

CDP8 lacks a simple normalise command. After =cdp8-sox-setup=, use:

#+begin_example
sox norm    — normalise to 0 dBFS (level param)
sox gain    — apply fixed dB gain
#+end_example

Both appear in the node menu and prompt field-by-field.

*** Stereo Panning and Imaging

#+begin_example
CDP8 pan:         abfpan, abfpan2   (B-format spatial)
CDP8 hard pan:    modify space
sox channels      — mono to stereo conversion
sox remix (E)     — manual: sox in.wav out.wav remix 1 0   (left channel only)
#+end_example

** Dynamics Processing

*** Compression and Limiting (SoX)

CDP8 has no compressor. After =cdp8-sox-setup=, use =sox compand=
which prompts for attack/decay and the transfer curve. For
complex curves use =E= to enter the full argument string:

#+begin_example
Soft compression (3:1 from -20dB):
  attack,decay  =  0.01,0.3
  transfer      = -90,-90,-30,-20,-20,-15,0,-5
  gain-in-dB    = -5

Hard limiter at -0.5dBFS:
  sox input.wav output.wav gain -h 0.5

Noise gate:
  sox input.wav output.wav silence 1 0.1 -50d 1 0.1 -50d
#+end_example

The SoX =compand= transfer curve format is: pairs of =input-dB:output-dB=,
comma-separated, from quietest to loudest.

*** Envelope Shaping (CDP8)

#+begin_example
Shape amplitude envelope:
  envel attack    (add attack ramp)
  envel curtail   (add release fade)
  envel dovetail  (crossfade with another file)
  envel impose    (apply envelope from a breakpoint file)
  modify loudness (overall level with peak normalisation)
#+end_example

To create a breakpoint envelope file for =envel impose=, when prompted
select =n= (new file) and write time/level pairs in the compose window:
#+begin_example
  0.0  0.0
  0.5  1.0
  3.5  1.0
  4.0  0.0
#+end_example

** Equalisation

*** CDP8 Filters

#+begin_example
Lo/hi pass (resonant):
  filter lohi   (lofrq, hifrq, resonance)

Parametric-style bank:
  filter userbank   (text file of centre/bandwidth/gain triplets)

Swept filter:
  filter sweeping   (centre freq sweeps between two values)
  filter variable   (breakpoint-driven filter)

Spectral EQ (operates on .ana files):
  hilite band   (boost a frequency region)
  hilite greq   (graphical EQ — text file of gains per channel)
  spec gain     (global spectral gain)
#+end_example

*** SoX EQ

#+begin_example
Shelving:
  sox bass   — low shelf at 100 Hz
  sox treble — high shelf at 3 kHz

Parametric band:
  sox equalizer — frequency, Q, gain (dB)

High/low pass:
  sox highpass, sox lowpass
#+end_example

All available field-by-field after =cdp8-sox-setup=.

** Reverb and Space

*** CDP8 Reverb

#+begin_example
Reverb-echo (simple):
  modify revecho   (delay, feedback, wetlevel)

Room simulation (multi-channel):
  modify space     (B-format spatial placement)
#+end_example

*** SoX Reverb

=sox reverb= after =cdp8-sox-setup= prompts for: reverberance (%), HF
damping (%), room scale (%), stereo depth (%), pre-delay (ms), wet gain (dB).

For impulse-response convolution reverb, render the signal from CDP8/SoX
and re-import it as a source node; convolve externally with jconvolver or
similar.

** Arranging and Timeline Assembly

*** Building a Timeline with CDP8

CDP8's =submix= family is the closest equivalent to a DAW timeline.

#+begin_example
1. Create individual processed clips: each is a node in the session.

2. Place clips at specific times:
   submix atstep    (uniform spacing — two files + step time)
   submix addtomix  (add single clip with position offset)

3. Balance levels:
   submix attenuate  (per-entry gain from breakpoint file)
   submix faders     (breakpoint-driven level automation)

4. Render the mix:
   combine sum   (sum all inputs)
   combine mean  (normalised sum to avoid clipping)
#+end_example

*** Trimming and Padding

#+begin_example
Cut a region (seconds):
  sox trim   (start and end times via schema prompts)

Add silence before/after:
  sox pad    (before-seconds, after-seconds)

Remove leading/trailing silence (CDP8):
  topantail2 topantail   (top-and-tail with threshold)
#+end_example

*** CDP8 Arrangement Commands

#+begin_example
Join multiple files end-to-end:
  combine make   (multi-file concatenation with pitch/formant data)
  combine make2  (with crossfade + envelope support)

Interleave into multi-channel:
  combine interleave

Insert silence at a point:
  repitch insertsil  (insert silence of given duration)

Generate silence:
  synth silence      (N seconds of silence as a wave node)
#+end_example

** Drum Programming

CDP8 and SoX do not have a step sequencer. Drum programming is best
approached as a timeline assembly problem:

*** Method A: Assemble Individual Samples

#+begin_example
1. Prepare drum samples (kick.wav, snare.wav, hihat.wav, etc.) as
   generator nodes pointing to your sample files, or process them
   with CDP8 effects (pitch tune, distort, grain grev, etc.)

2. Create a timeline file manually with submix atstep or addtomix,
   placing each hit at its precise time offset.

3. Render with combine sum.
#+end_example

A 4/4 kick on beats, 120 BPM = interval of 0.5s:
#+begin_example
  submix atstep kick.wav kick.wav   (step=0.5 for 4 beats per pass)
  submix addtomix + snare.wav at 0.5, 1.5, 2.5 …
#+end_example

*** Method B: SoX Synthetic Drums

These are complex one-liners; enter them via =E= (raw command edit):
#+begin_example
Kick drum (tuned sine sweep):
  sox -n kick.wav synth 0.3 sine 150:60 fade 0 0.3 0.1 gain -5

Snare (short noise burst):
  sox -n snare.wav synth 0.2 whitenoise fade 0 0.2 0.05 gain -8

Hi-hat (short high-pass noise):
  sox -n hat.wav synth 0.05 whitenoise highpass 8000 fade 0 0.05 0 gain -12
#+end_example

*** Method C: Clicknew as a Rhythmic Skeleton

#+begin_example
[generator]  clicknew             → a1.wav
             (clicktimes_datafile lists each hit time in seconds)
#+end_example

Clicknew places a click at each specified time. These clicks can drive
CDP8 grain-based effects (grain reverse, grain repitch) to produce
rhythmic events from sample material.

** Format Conversion and Sample Rate

*** CDP8

#+begin_example
Channel conversion:
  housekeep chans  (change channel count in header)
  channelx         (extract single channel from multichannel file)
  combine interleave (mono files → multichannel)

Sample rate (header only, no resampling):
  housekeep respec  (change sample rate in header)
#+end_example

For actual resampling use SoX (=sox rate= after =cdp8-sox-setup=).

*** SoX Format Conversion

#+begin_example
Resample to 48 kHz:
  sox rate    (via schema prompts after cdp8-sox-setup)
  or via E: sox input.wav -r 48000 output.wav rate -v

Convert to stereo:
  sox channels 2

Convert bit depth:
  via E: sox input.wav -b 24 output.wav

Convert to AIFF (for import into other DAWs):
  via E: sox input.wav output.aiff
#+end_example

** Mastering Chain Example

A complete signal path from raw mix to master:

#+begin_example
  a1.wav   [raw rendered mix]

  → [effect] filter lohi        → a2.wav   (hi-pass at 30 Hz, remove rumble)
  → [effect] modify loudness    → a3.wav   (CDP8 level normalisation)
  → [SoX]    sox compand        → a4.wav   (2:1 compression, -18 dBFS threshold)
  → [effect] distort overload   → a5.wav   (mild saturation, gain=1.2)
  → [SoX]    sox equalizer      → a6.wav   (air boost: 5 kHz, Q=1, +1.5 dB)
  → [SoX]    sox reverb         → a7.wav   (subtle room: reverberance=15)
  → [SoX]    sox norm           → a8.wav   (true-peak limit to -0.3 dBFS)
#+end_example

Each SoX step is a schema-driven node created via =n= after =cdp8-sox-setup=.

* Keyboard Reference

|-----|--------|
| =n= | New node (prompts for type → input → command → params) |
| =RET= | Node actions: play / waveform / spectrogram / new effect |

| =R= | Run all pending nodes (topological order) |

shows the binary's USAGE text. Close it with =q= or =C-x 1=.

* SoX Quick Reference for CDP8 Users

After =cdp8-sox-setup=, these appear as schema-driven nodes via =n=.
For complex multi-effect chains use =E= to write the full command line.

| Task | SoX node / fragment |
|------|---------------------|
| Normalise | =sox norm= |
| Gain | =sox gain= |
| Fade in/out | =sox fade= |
| Trim | =sox trim= |
| Pad silence | =sox pad= |
| Reverse | =sox reverse= |
| Repeat N times | =sox repeat= |
| Tempo (no pitch) | =sox tempo= |
| Pitch shift (semitones) | =sox pitch= |
| Compress/expand | =sox compand= |
| Reverb | =sox reverb= |
| Lo-pass | =sox lowpass= |
| Hi-pass | =sox highpass= |
| Parametric EQ | =sox equalizer= |
| Remove silence | =sox silence= |
| Resample | =sox rate= |

Full SoX syntax for raw =E= editing:
#+begin_example
  sox infile outfile EFFECT [options] [EFFECT [options] ...]
#+end_example

Multiple effects can be chained in a single SoX invocation.
