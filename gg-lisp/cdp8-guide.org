* Overview

This guide covers three areas:

1. Workflow — how to use cdp8-mode effectively, from importing sources to
   running chains and auditioning results.
2. Experimental music recipes — 20 node chains for electronic and spectral
   music composition.
3. Standard DAW workflows — mixing, dynamics, EQ, reverb, arranging, drums,
   mastering.

The mode represents processing chains as typed cables.  Each node outputs a
file whose name encodes its type:

| Cable      | Type                              |
|------------+-----------------------------------|
| =a1.wav=   | audio (wave)                      |
| =s1.ana=   | spectral analysis (phase vocoder) |
| =p1.frq=   | binary pitch data                 |
| =b1.brk=   | breakpoint (time/value text)      |
| =e1.env=   | binary envelope data              |

When you select an effect from a node's =RET= menu, only commands that accept
that cable type as input are offered.

** SoX integration

Load the optional SoX module once (add to init or eval in =*scratch*=):

#+begin_src emacs-lisp
(require 'cdp8-sox)
(cdp8-sox-setup)
#+end_src

After setup, 21 SoX effects appear in the command menu exactly like CDP8
effects, with field-by-field parameter prompts.

** Text data files

Many CDP8 parameters are text files (breakpoint envelopes, timing lists,
filter tables, …).  When prompted, choose:

- =y= — pick an existing file
- =n= — open a compose window pre-populated with a format comment;
  =C-c C-c= to confirm, =C-c C-q= to cancel

The file is saved to the session directory.

* Workflow

** Starting a session

#+begin_example
M-x cdp8-session RET /path/to/project-dir/ RET
#+end_example

All output files are written there.  The session is persisted as
=cdp8-session.el= in that directory and reloaded automatically.

** Importing source material

CDP8's built-in generators (=synth wave=, =synth noise=, =chirikov=, …) are
useful for purely synthetic work but spectrally simple.  Real recordings —
field sounds, voice, instruments, found audio — are far richer sources for
spectral and granular processing.

Press =i= to open the import menu:

| Key | Action |
|-----+--------|
| =y= | Download from YouTube (requires =yt-dlp=) |
| =f= | Import a local audio file |

*** YouTube download

Press =i y=, then enter a YouTube URL or bare video ID:

#+begin_example
  https://www.youtube.com/watch?v=dQw4w9WgXcQ
  https://youtu.be/dQw4w9WgXcQ
  dQw4w9WgXcQ
#+end_example

The audio is downloaded as a WAV file named =<video-id>.wav= in the session
directory and immediately registered as a source node (status: done).
Install yt-dlp if needed: =nix-shell -p yt-dlp=.

*** Local file import

Press =i f= and choose a WAV file.  It is copied into the session directory
and registered as a source node.

*** What makes a good source?

The more complex and varied the spectrum, the richer the results from
spectral and granular processing:

- *Sustained sounds with internal motion*: bowed strings, wind, breath,
  resonant metal — phase vocoder stretching reveals micro-fluctuations.
- *Percussive textures*: rain, gravel, crowd — grain operations create
  unpredictable rhythmic clouds.
- *Speech and voice*: formant-rich, highly structured; vowel and pitch
  processing produce singing/morphing effects.
- *Dense chords and noise*: spectral subtraction and blurring find hidden
  structure inside noise.
- *Avoid*: pure sine waves and simple synth tones are too sparse to yield
  interesting spectral results.

** Building a processing chain

The typical workflow:

1. Import or generate a source → =a1.wav= appears with status *done*
2. Press =RET= on the source node → choose =n= (new effect)
3. Select an effect from the completing-read (filtered by cable type)
4. Fill in parameters at the prompts
5. Press =r= to run the effect node
6. Press =p= to play the result immediately; =RET= for play/waveform/spectrogram
7. Press =e= to re-edit parameters, =r= again to re-run

For spectral work, the standard chain is:

#+begin_example
  source.wav → pvoc anal → s1.ana → [spectral effects] → pvoc synth → a2.wav
#+end_example

The =pvoc anal= / =pvoc synth= pair wraps any chain of =.ana= effects.

** Parameter editing

- =e= — re-prompts all parameters; current values appear as defaults
- =E= — edit the raw command string (for complex SoX one-liners or
  free-form commands without a schema)
- For number params, just press =RET= to accept the shown default
- For breakpoint/data files, choose =n= to open the compose window and write
  or edit the data inline

** Running nodes

- =r= — run the node at point (marks it *running*, then *done* or *error*)
- =R= — run all pending nodes in topological order (respects dependencies)
- Check the =*cdp8-run: <id>*= buffer for stdout/stderr if a node errors

** Auditioning

- =p= — play the output of the node at point (instant, no menu)
- =RET= → =p= — same but from the action menu
- =RET= → =w= — open a waveform image (requires ffmpeg)
- =RET= → =s= — open a spectrogram (requires sox)

** Iterating and branching

Nodes are not modified in place — re-editing creates a new version.  To
branch (try a different parameter on the same source):

1. Press =RET= on the source node → =n= new effect
2. Select the same effect with different parameters
3. Both effect nodes point to the same input; run each independently

To discard a failed attempt: =d= on the node.  The output file is optionally
deleted.

* Experimental Music: 20 Composition Recipes

Source material suggestions are given for each recipe.  Press =i= to import.
The =synth wave= alternatives shown in brackets are fallbacks for when no
real source is available.

** 1. Spectral Freeze Drone

*Best source*: a bowed string, singing bowl, or sustained wind instrument —
any sound with slow internal motion.

#+begin_example
  [source]     field recording or instrument   → a1.wav   (or: synth wave)
  [effect]     pvoc anal                       → s1.ana
  [effect]     spec grab      (time=0.5)       → s2.ana
  [effect]     spec magnify   (dur=30.0)       → s3.ana
  [effect]     pvoc synth                      → a2.wav
#+end_example

=spec grab= freezes a single analysis frame; =spec magnify= stretches it to
any duration.  Vary =time= to select different spectral moments.

** 2. Granular Cloud Diffusion

*Best source*: speech, crowd noise, or a textured field recording — anything
with grain-level variation.

#+begin_example
  [source]     speech or crowd recording       → a1.wav
  [effect]     grain r_extend   (time=10.0)    → a2.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     blur scatter     (keep=2)       → s2.ana
  [effect]     pvoc synth                      → a3.wav
#+end_example

Granular diffusion converted into a smeared spectral texture.

** 3. Stochastic Spectral Walk

*Best source*: any rich recording; the more internal variation, the more
unpredictable the walk.

#+begin_example
  [source]     forest, rain, or resonant metal → a1.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     blur drunk   (range=8, dur=20)  → s2.ana
  [effect]     pvoc synth                      → a2.wav
#+end_example

=blur drunk= takes a random walk through time-windows of the spectrum.
Each run produces a different result.

** 4. Cross-Synthesis (Spectral Hybridisation)

*Best source*: two contrasting sounds — one with clear attacks (percussion,
speech) and one with sustained spectrum (strings, choir, wind).

#+begin_example
  [source A]   voice or percussion             → a1.wav
  [source B]   bowed strings or choir          → a2.wav
  [effect]     pvoc anal  (a1)                 → s1.ana
  [effect]     pvoc anal  (a2)                 → s2.ana
  [effect]     specross   (s1, s2)             → s3.ana
  [effect]     pvoc synth                      → a3.wav
#+end_example

The rhythm and attack structure of A is coloured by the spectrum of B.

** 5. Harmonic Distortion Chain

*Best source*: a single sustained note — instrument, voice, or synth sine.
Richer harmonics emerge from simple sources.

#+begin_example
  [source]     single sustained note           → a1.wav   (or: synth wave)
  [effect]     distort harmonic                → a2.wav
               (harmonics-file: 1 1.0 / 2 0.5 / 3 0.25 / 4 0.1)
  [effect]     filter lohi  (lo=200, hi=4000)  → a3.wav
  [effect]     modify loudness                 → a4.wav
#+end_example

** 6. Spectral Gate and Freeze

*Best source*: a dense chord, gong strike, or any spectrally rich transient.

#+begin_example
  [source]     gong or dense chord             → a1.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     spec gate      (threshold=0.02) → s2.ana
  [effect]     hilite band    (centrefrq=440)  → s3.ana
  [effect]     spec magnify   (dur=8.0)        → s4.ana
  [effect]     pvoc synth                      → a2.wav
#+end_example

The gate removes quiet partials; =hilite band= isolates a region; =magnify=
sustains the surviving spectrum.

** 7. Granular Repitch Scatter

*Best source*: a textured, grainy sound — crumpled paper, gravel, rain.

#+begin_example
  [source]     textured percussive sound       → a1.wav
  [effect]     grain repitch                   → a2.wav
               (transpmultfile: random ratio pairs)
  [effect]     distort average   (cyclecnt=4)  → a3.wav
  [effect]     filter phasing                  → a4.wav
#+end_example

** 8. Brownian Frequency Motion

Purely synthetic: the generator creates a tone that drifts randomly in
frequency.

#+begin_example
  [generator]  brownian motion                 → a1.wav
               (mode=1, plo=80, phi=1200, step=50, dur=20)
  [effect]     distort envel                   → a2.wav
  [effect]     filter variable                 → a3.wav
#+end_example

=plo= and =phi= set the frequency range; =step= controls drift speed.

** 9. Cantor Set Rhythmic Gating

*Best source*: a sustained drone or pad — the recursive silences are most
audible against a continuous background.

#+begin_example
  [source]     sustained drone                 → a1.wav   (or: synth wave)
  [effect]     cantor set   (holesize=0.5, depth-trig=4) → a2.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     blur chorus                     → s2.ana
  [effect]     pvoc synth                      → a3.wav
#+end_example

=cantor set= punches recursive silence holes into the audio before the
spectral blurring blurs the edges.

** 10. Spectral Sharpening via Focus Accumulation

*Best source*: any pitched source with multiple partials — piano, guitar,
or voice.

#+begin_example
  [source]     piano or guitar note            → a1.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     focus accu   (acuity=6.0)       → s2.ana
  [effect]     blur avrg    (N=7)              → s3.ana
  [effect]     pvoc synth                      → a2.wav
#+end_example

=focus accu= concentrates energy at spectral peaks; =blur avrg= then
softens the sharpened result into a metallic resonance.

** 11. Iterative Distort Reform

*Best source*: a complex waveform or voice; the cyclic distortion accumulates
in proportion to the source's harmonic richness.

#+begin_example
  [source]     voice or complex instrument     → a1.wav   (or: synth wave mode=3)
  [effect]     distort reform   (mode=2)       → a2.wav
  [effect]     distort reform   (mode=3)       → a3.wav
  [effect]     filter sweeping                 → a4.wav
#+end_example

Each =distort reform= pass re-shapes waveform cycles differently.

** 12. Click Rhythm Machine

Purely algorithmic: =clicknew= places clicks at exact times from a data file.

#+begin_example
  [generator]  clicknew                        → a1.wav
               (clicktimes_datafile: beat times in seconds, one per line)
               Example for 120 BPM 4/4: 0.0 / 0.5 / 1.0 / 1.5 / 2.0 / …
  [effect]     grain reverse    (len=0.05)     → a2.wav
  [effect]     modify revecho                  → a3.wav
#+end_example

Grain reversal turns clicks into small granular pops; revecho builds a wash.

** 13. Waveform Sculpting (Spectral Noise Injection)

*Best source*: voice or a harmonically rich instrument.

#+begin_example
  [source]     voice or strings                → a1.wav   (or: synth wave)
  [effect]     pvoc anal                       → s1.ana
  [effect]     blur noise   (noise=0.3)        → s2.ana
  [effect]     spec gain    (gain=1.5)         → s3.ana
  [effect]     pvoc synth                      → a2.wav
#+end_example

Spectral noise injection roughens the partials without destroying pitch.

** 14. Granular Time Reversal Cloud

*Best source*: speech or any source with clear directionality — the reversal
effect is most striking when direction matters.

#+begin_example
  [source]     speech or spoken word           → a1.wav
  [effect]     grain reverse   (len=0.1)       → a2.wav
  [effect]     grain reverse   (len=0.3)       → a3.wav
  [effect]     combine mean   (a2, a3)         → a4.wav
#+end_example

Two grain-reversed streams at different grain sizes averaged together.
Produces ambiguous temporal directionality.

** 15. Crystal Resonator

*Best source*: a sound with clear harmonic content that will survive phase
rotation — bowed metal, cello harmonics.

#+begin_example
  [source]     bowed metal or cello            → a1.wav
  [effect]     crystal rotate                  → a2.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     focus fold   (lofrq=100, hifrq=1000) → s2.ana
  [effect]     pvoc synth                      → a3.wav
#+end_example

Crystal rotation shifts partials in phase; focus folding folds the high
partials back down, compressing the spectrum.

** 16. Spectral Blur Depth Map

*Best source*: any rich spectrum; contrast between near and far blur is
sharpest with dense harmonic content.

#+begin_example
  [source]     choir or orchestra              → a1.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     blur blur  (s1, blurring=2)     → s2.ana
  [effect]     blur blur  (s1, blurring=20)    → s3.ana
  [effect]     specnu subtract  (s2, s3)       → s4.ana
  [effect]     pvoc synth                      → a2.wav
#+end_example

Subtracting heavily-blurred from lightly-blurred isolates mid-timescale
spectral motion — a kind of spectral "edge detection" in time.

** 17. Fractal Spectral Distortion

*Best source*: a clean percussive or instrumental sound where the added
self-similar copies are audible as texture.

#+begin_example
  [source]     piano or plucked string         → a1.wav
  [effect]     distort fractal  (scaling=4)    → a2.wav
  [effect]     pvoc anal                       → s1.ana
  [effect]     spec gate   (threshold=0.01)    → s2.ana
  [effect]     pvoc synth                      → a3.wav
#+end_example

Fractal distortion adds self-similar waveform copies at different scales.
Spectral gating removes the noise floor introduced by the distortion.

** 18. Ceracu Cycle Morphology

*Best source*: a short pitched sound with clear cycles — a single instrument
note, a vowel, a synth tone.  Ceracu needs identifiable waveform cycles.

#+begin_example
  [source]     single flute or clarinet note   → a1.wav
  [effect]     ceracu ceracu                   → a2.wav
               (cyclcnts=10, mincycdur=0.1, outdur=20.0)
  [effect]     strans multi   (speed=0.5)      → a3.wav
  [effect]     modify space                    → a4.wav
#+end_example

Ceracu stretches each waveform cycle individually — a form of additive
time-stretching that preserves the timbral texture of a single cycle.

** 19. Chirikov Map Modulator

Purely synthetic: Chirikov generates chaotic audio from a standard map.

#+begin_example
  [generator]  chirikov chirikov               → a1.wav
               (mode=1, frq=200, damping=0.5, dur=10.0)
  [effect]     distort multiply  (a1, a1)      → a2.wav   (self ring-mod)
  [effect]     pvoc anal                       → s1.ana
  [effect]     blur scatter   (keep=2)         → s2.ana
  [effect]     pvoc synth                      → a3.wav
#+end_example

Chaotic frequency trajectories self-ring-modulated, then spectrally scattered.

** 20. Formant Singing Synthesis

*Best source*: a harmonic generator suffices here; =synth spectra= builds the
harmonic stack that the vowel formants then shape.

#+begin_example
  [generator]  synth spectra                   → a1.wav
               (harmonic series, duration=4.0)
  [effect]     repitch vowels                  → a2.wav
               (vowel-data: "a e i o u" sequence)
  [effect]     modify loudness                 → a3.wav
  [effect]     modify revecho                  → a4.wav
#+end_example

Harmonic source shaped by vocal formants over time, giving the impression of
a slowly morphing singing voice.

* Standard DAW Workflows

** Mixing

*** Basic Stereo Mix (CDP8)

#+begin_example
Create a mix file:
  submix atstep   (two inputs + step time → outmixfile)

Add tracks:
  submix addtomix (add another file at a given time offset)

Apply automation:
  submix faders   (breakpoint file: time/level pairs)

Render:
  combine sum     (sum all inputs)
  combine mean    (normalised sum — avoids clipping on dense mixes)
#+end_example

*** Gain and Normalisation

#+begin_example
CDP8:   modify loudness   (peak normalisation)
SoX:    sox norm          (normalise to target dBFS, field-by-field)
SoX:    sox gain          (fixed dB offset)
#+end_example

*** Stereo and Spatial

#+begin_example
CDP8 B-format:   abfpan, abfpan2   (full 3-D spatial pan)
CDP8 simple:     modify space      (stereo placement)
SoX:             sox channels 2    (mono → stereo duplication)
#+end_example

** Dynamics

*** Compression (SoX)

=sox compand= after =cdp8-sox-setup= prompts for attack/decay and the
transfer curve.  For complex curves, use =E= and enter manually:

#+begin_example
Soft compression (3:1, threshold -20 dB):
  attack,decay  = 0.01,0.3
  transfer      = -90,-90,-30,-20,-20,-15,0,-5
  gain-in-dB    = -5

Noise gate:
  sox input.wav output.wav silence 1 0.1 -50d 1 0.1 -50d
#+end_example

*** Envelope Shaping (CDP8)

#+begin_example
envel attack    — add attack ramp
envel curtail   — add release fade
envel dovetail  — crossfade with another file
envel impose    — apply any amplitude shape from a breakpoint file
modify loudness — overall level with peak normalisation
#+end_example

When prompted for a breakpoint file (=envel impose=), choose =n= (new file)
and write time/level pairs in the compose window.

** Equalisation

*** CDP8 Filters

#+begin_example
filter lohi      — resonant lo/hi pass (lofrq, hifrq, resonance)
filter userbank  — text file of centre/bandwidth/gain triplets
filter sweeping  — frequency sweep between two values
filter variable  — breakpoint-driven cutoff
#+end_example

*** Spectral EQ (on =.ana= files)

#+begin_example
hilite band  — boost a frequency region
hilite greq  — graphical EQ (text file: one gain per channel)
spec gain    — global spectral level
spec gate    — zero partials below threshold
#+end_example

*** SoX EQ

#+begin_example
sox bass, sox treble      — shelving filters
sox equalizer             — parametric band (freq, Q, dB)
sox highpass, sox lowpass — cut filters
#+end_example

All available field-by-field after =cdp8-sox-setup=.

** Reverb and Space

#+begin_example
CDP8:   modify revecho   — reverb-echo (delay, feedback, level)
CDP8:   modify space     — B-format spatial placement
SoX:    sox reverb       — Freeverb (reverberance %, HF-damp %, room-scale %)
#+end_example

** Arranging and Timeline Assembly

#+begin_example
1. Each processed clip is a node; run all with R.
2. Place clips: submix atstep / submix addtomix
3. Automate levels: submix faders (breakpoint file)
4. Trim edges: sox trim / topantail2 topantail
5. Add gaps: sox pad (before-s, after-s)
6. Concatenate: combine make / combine make2 (with crossfade)
7. Render: combine sum or combine mean
#+end_example

** Drum Programming

*** Method A: Sample Assembly

#+begin_example
1. Import drum samples with i f (local file import).
2. Process each with CDP8 (pitch tune, distort, grain grev, …).
3. Place with submix atstep / addtomix at beat times.
4. Render with combine sum.
#+end_example

At 120 BPM, 4/4 beat interval = 0.5 s.

*** Method B: SoX Synthetic Drums (via =E=)

#+begin_example
Kick:   sox -n kick.wav synth 0.3 sine 150:60 fade 0 0.3 0.1 gain -5
Snare:  sox -n snare.wav synth 0.2 whitenoise fade 0 0.2 0.05 gain -8
Hi-hat: sox -n hat.wav synth 0.05 whitenoise highpass 8000 fade 0 0.05 0 gain -12
#+end_example

Create a source node with =n=, then edit its command with =E=.

*** Method C: Clicknew + Grain Processing

#+begin_example
[generator]  clicknew      → a1.wav   (click times from a data file)
[effect]     grain repitch → a2.wav   (scatter clicks across pitches)
[effect]     combine sum   → a3.wav   (mix multiple passes)
#+end_example

** Format Conversion and Sample Rate

#+begin_example
SoX resampling (preferred):
  sox rate   (schema node: choose target sample rate)

Channel conversion:
  sox channels   (schema node)
  channelx       (extract one channel from multichannel)
  combine interleave   (mono files → multichannel)

Header-only SR change (no resampling):
  housekeep respec
#+end_example

** Mastering Chain Example

#+begin_example
  a1.wav   [raw rendered mix]
  → filter lohi        → a2.wav   (hi-pass 30 Hz, remove rumble)
  → modify loudness    → a3.wav   (CDP8 peak normalise)
  → sox compand        → a4.wav   (light compression)
  → distort overload   → a5.wav   (tape saturation, gain ≈ 1.2)
  → sox equalizer      → a6.wav   (air: 5 kHz, Q=1, +1.5 dB)
  → sox reverb         → a7.wav   (subtle room)
  → sox norm           → a8.wav   (peak limit to -0.3 dBFS)
#+end_example

Each SoX step is a schema-driven node via =n= after =cdp8-sox-setup=.
